Crea una app web llamada "C4S Scout" para speakers técnicos de comunidades GitHub y Microsoft.

Objetivo: detectar Call for Speakers activos, controlar deadlines, organizar propuestas de charla y hacer seguimiento de candidaturas.

Pantallas:
1. Dashboard con KPIs: C4S activos, deadlines próximos, propuestas en borrador, candidaturas enviadas.
2. Lista de oportunidades con filtros por comunidad, fecha límite, país, formato y tecnología.
3. Pipeline de candidaturas con estados: Idea, Draft, Ready, Submitted, Accepted, Rejected.
4. Biblioteca de propuestas con título, abstract, nivel, audiencia y tags.

Reglas:
- Priorizar deadlines a 7, 14 y 30 días.
- Marcar oportunidades sin propuesta asociada como riesgo si quedan menos de 7 días.
- Incluir datos de ejemplo realistas.
- UI moderna, oscura, clara para una demo de 45 minutos.
