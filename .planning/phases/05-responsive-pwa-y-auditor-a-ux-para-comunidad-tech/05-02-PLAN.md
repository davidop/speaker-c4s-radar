## Task 05-02-03: Crear service worker sw.js

**type:** implementation
**files:** demo/public/sw.js

**action:**
Crear `demo/public/sw.js` con estrategia cache-first para assets estáticos y datos:

```javascript
const CACHE_NAME = 'c4s-radar-v1';
const ASSETS_TO_CACHE = [
  '/',
  '/index.html',
  // Los assets de Vite tienen hashes dinámicos; cachear en runtime
];

// Install: pre-cache shell
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => cache.addAll(ASSETS_TO_CACHE))
  );
  self.skipWaiting();
});

// Activate: limpiar caches antiguas
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((keys) =>
      Promise.all(keys.filter((k) => k !== CACHE_NAME).map((k) => caches.delete(k)))
    )
  );
  self.clients.claim();
});

// Fetch: cache-first con fallback a network
self.addEventListener('fetch', (event) => {
  // Solo manejar requests GET del mismo origen
  if (event.request.method !== 'GET') return;
  
  event.respondWith(
    caches.match(event.request).then((cached) => {
      if (cached) return cached;
      return fetch(event.request).then((response) => {
        // Cachear assets estáticos y datos JSON
        if (
          response.ok &&
          (event.request.url.includes('/assets/') ||
           event.request.url.endsWith('.json') ||
           event.request.url.endsWith('.css') ||
           event.request.url.endsWith('.js'))
        ) {
          const clone = response.clone();
          caches.open(CACHE_NAME).then((cache) => cache.put(event.request, clone));
        }
        return response;
      });
    })
  );
});
```

**verify:** El archivo existe y no tiene errores de sintaxis JS.
**acceptance_criteria:** `sw.js` tiene handlers para `install`, `activate` y `fetch`.

## Task 05-02-04: Registrar service worker en main.js

**type:** implementation
**files:** demo/src/main.js

**action:**
Añadir al final de `demo/src/main.js` (antes del `render()` inicial o al final del archivo) el registro del service worker:

```javascript
// Registro PWA
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/sw.js').catch((err) => {
      console.warn('SW registration failed:', err);
    });
  });
}
```

**Nota:** Si el base path en producción es `/speaker-c4s-radar/`, la ruta del SW debe ser `/speaker-c4s-radar/sw.js`. Usar el mismo mecanismo que `vite.config.js` usa para el `base`. Por ahora usar `/sw.js` que funciona en desarrollo; Vite copiará el SW desde `public/` automáticamente.

**verify:** El bloque de registro aparece en main.js y no rompe la funcionalidad existente.
**acceptance_criteria:** `navigator.serviceWorker.register` está presente en main.js con guard `'serviceWorker' in navigator`.
