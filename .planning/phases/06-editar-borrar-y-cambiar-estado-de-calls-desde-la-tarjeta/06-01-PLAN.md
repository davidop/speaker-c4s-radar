## Task 06-01-01: Extender API local para update/delete

**type:** implementation
**files:** demo/vite.config.js

**action:**
Refactorizar el middleware `'/api/calls'` para soportar:
- `POST /api/calls` (actual, mantener)
- `PATCH /api/calls/:id` para cambio de estado y updates parciales
- `DELETE /api/calls/:id` para borrado

Puntos de implementación:
- Parsear `req.url` y extraer `id` si existe.
- Leer `src/data/calls.json`, localizar registro por `id`.
- `PATCH`: merge parcial (campos permitidos: `name, community, deadline, city, format, tags, status, proposalId, audience, source`).
- `DELETE`: remover el registro, devolver 200 con `{ deleted: true, id }`.
- Errores: 404 si no existe id; 400 payload inválido; 500 fallo IO.

**verify:** Con `curl` local, `PATCH` y `DELETE` devuelven códigos esperados.
**acceptance_criteria:** API local soporta update/delete sin romper create.

## Task 06-01-02: Añadir acciones inline en card

**type:** implementation
**files:** demo/src/main.js

**action:**
En la función `card(call)` añadir:
- `select` de estado con opciones `stateOrder` y valor actual.
- Botón `Editar`.
- Botón `Borrar`.

Exponer handlers globales:
- `window.updateCallStatus(id, status)`
- `window.deleteCall(id)`

Flujo `updateCallStatus`:
- `fetch('/api/calls/{id}', { method: 'PATCH', ... })`
- actualizar `callsData` en memoria con respuesta
- `render()`
- fallback alert si falla (modo estático)

Flujo `deleteCall`:
- pedir confirmación `confirm(...)`
- `fetch('/api/calls/{id}', { method: 'DELETE' })`
- filtrar `callsData`
- `render()`
- fallback alert si falla

**verify:** Cambio de estado y borrado se reflejan en tarjetas, KPIs y pipeline.
**acceptance_criteria:** Usuario puede cambiar estado/borrar desde cualquier card.
