## Task 06-02-01: Añadir modo edición en frontend

**type:** implementation
**files:** demo/src/main.js

**action:**
Introducir estado de edición:
- `let editingCallId = null;`

Cambios en render/form:
- Si `editingCallId` existe, pre-rellenar formulario con datos del call.
- Cambiar CTA del botón a `Guardar cambios`.
- Mostrar botón secundario `Cancelar edición`.

Nuevo handler:
- `window.startEditCall(id)` para activar edición desde cada card.
- Reusar `window.addCall` como submit unificado:
  - si `editingCallId` es null -> `POST /api/calls`
  - si existe -> `PATCH /api/calls/{id}`

Al guardar edición:
- actualizar `callsData` con respuesta
- limpiar `editingCallId`
- reset form
- re-render

**verify:** Al pulsar Editar en una card, el formulario muestra datos actuales y permite guardar.
**acceptance_criteria:** Usuario puede editar campos principales sin recargar página.

## Task 06-02-03: Endurecer validación PATCH en middleware

**type:** implementation
**files:** demo/vite.config.js

**action:**

- Rechazar payload vacío.
- Validar `status` contra `['Idea','Draft','Ready','Submitted','Accepted','Rejected']` si viene en payload.

**verify:** Requests inválidos devuelven 400 con mensaje útil.
**acceptance_criteria:** API update no permite estados/fechas inválidas.
